# 同主题新闻合并功能

## 📖 功能概述

当选题结果包含多条相同主题的新闻时（如"小米汽车发布"、"小米汽车订单"、"小米汽车销量"），自动将它们合并为一条复合新闻，并在脚本生成时以**递进式叙述**串联。

---

## 🎯 解决的问题

### **问题场景**

用户观察到运行结果：
```
选中的新闻：
1. 小米汽车 SU7 发布新车型
2. 小米汽车订单突破 10 万
3. 小米汽车销量增长 200%
```

**当前行为**：3 条新闻独立叙述，重复感强，听感割裂。

**期望行为**：合并为 1 条复合新闻，递进式叙述：
```
"小米汽车最近动作频频。先是发布了新车型 SU7...
紧接着订单数据也传来好消息，突破了 10 万辆...
而最新的销量数据更是亮眼，同比增长 200%..."
```

---

## ⚙️ 工作原理

### **1. 同主题检测**

使用 3 种策略判断新闻是否相关：

#### **策略 1：实体重叠度**
```python
# 实体相似度 = 交集 / 并集
entities1 = {"小米汽车", "雷军", "SU7"}
entities2 = {"小米汽车", "订单", "销量"}

intersection = {"小米汽车"}
union = {"小米汽车", "雷军", "SU7", "订单", "销量"}

similarity = 1 / 5 = 0.2
```

阈值：`entity_similarity_threshold = 0.5`（默认）

#### **策略 2：标题相似度**
```python
title1 = "小米汽车 SU7 发布新车型"
title2 = "小米汽车订单突破 10 万"

# 使用 SequenceMatcher 计算相似度
similarity = 0.45  # 有"小米汽车"这个共同前缀
```

阈值：`title_similarity_threshold = 0.3`（默认）

#### **策略 3：共享关键词**
```python
keywords1 = {"小米汽车", "SU7", "新车型"}
keywords2 = {"小米汽车", "订单", "10万"}

shared = {"小米汽车"}
```

阈值：`min_shared_keywords = 2`（默认）

**满足任意 1 个策略即判定为同主题。**

---

### **2. 聚类合并**

使用深度优先搜索（DFS）将相关新闻聚类：

```
新闻 A ─── 相关 ─── 新闻 B
          ╱
新闻 C ───
          ╲
新闻 D ─── 相关 ─── 新闻 E

结果：
- 聚类 1: [A, B, C]
- 聚类 2: [D, E]
```

---

### **3. 合并为复合新闻**

合并后的数据结构：

```python
{
    "id": "merged_xiaomi_auto_001",
    "title": "小米汽车系列动态（3条）",
    "text": """
【动态1】小米汽车 SU7 发布新车型...
【动态2】小米汽车订单突破 10 万...
【动态3】小米汽车销量增长 200%...
    """,
    "source_name": "36氪、界面新闻 等3家媒体",
    "entities": ["小米汽车", "雷军", "SU7", "订单", "销量"],
    
    "is_merged": True,
    "merged_count": 3,
    "sub_items": [原始新闻1, 原始新闻2, 原始新闻3],
    "narrative_hint": "三段式叙述：按时间线或逻辑顺序依次展开，形成完整的发展脉络"
}
```

---

### **4. 递进式叙述提示**

传递给 LLM 的 prompt 增强：

```
【新闻列表】
1. 小米汽车系列动态（3条） [合并新闻×3]
   （36氪、界面新闻 等3家媒体）
   摘要：【动态1】小米汽车 SU7 发布新车型...
         【动态2】小米汽车订单突破 10 万...
         【动态3】小米汽车销量增长 200%...
   
   ⚠️ 叙述提示：三段式叙述：按时间线或逻辑顺序依次展开，形成完整的发展脉络
   
   包含以下子新闻：
      1) 小米汽车 SU7 发布新车型
      2) 小米汽车订单突破 10 万
      3) 小米汽车销量增长 200%
```

LLM 会根据 `narrative_hint` 生成递进式脚本。

## 📝 使用方式

### **配置文件** `config/base/settings.yaml`

```yaml
script:
  news_merger:
    enabled: true                       # 是否启用新闻合并
    entity_similarity_threshold: 0.5    # 实体重叠度阈值（0-1）
    title_similarity_threshold: 0.3     # 标题相似度阈值（0-1）
    min_shared_keywords: 2              # 最少共享关键词数
```

### **调整阈值**

#### **更严格匹配**（减少误合并）
```yaml
entity_similarity_threshold: 0.7    # 更高的实体重叠要求
title_similarity_threshold: 0.5     # 更高的标题相似度
min_shared_keywords: 3              # 更多的共享关键词
```

#### **更宽松匹配**（合并更多相关新闻）
```yaml
entity_similarity_threshold: 0.3
title_similarity_threshold: 0.2
min_shared_keywords: 1
```

#### **完全禁用**
```yaml
script:
  news_merger:
    enabled: false
```

## 🛠️ 技术实现

### **核心模块**

| 模块 | 文件 | 功能 |
|------|------|------|
| 合并器 | `src/script/news_merger.py` | 同主题检测、聚类、合并 |
| 集成点 | `src/app/pipelines/steps/script_step_segmented.py` | 脚本生成前执行合并 |
| Prompt 增强 | `src/llm/segment_generator.py` | 添加合并新闻标注和叙述提示 |

### **数据流**

```
选题结果 (3 条小米汽车新闻)
    ↓
NewsMerger.merge_news_items()
    ├─ _build_similarity_graph()      # 计算相似度
    ├─ _cluster_by_similarity()        # DFS 聚类
    └─ _merge_cluster()                # 合并为复合新闻
    ↓
合并结果 (1 条复合新闻 + 叙述提示)
    ↓
_format_news_items_for_prompt()        # 格式化为 LLM prompt
    ├─ 添加 [合并新闻×3] 标注
    ├─ 展开子新闻列表
    └─ 注入叙述提示
    ↓
LLM 生成递进式脚本
```

## ⚠️ 注意事项

### **1. 实体提取质量**

合并依赖实体提取，如果实体质量差，合并效果也会受影响。

**解决**：优化 `signal_tagging` 的实体提取 prompt。

### **2. 过度合并**

阈值设置过低可能导致不相关新闻被误合并。

**解决**：提高 `entity_similarity_threshold` 或 `min_shared_keywords`。

### **3. LLM 未遵循叙述提示**

LLM 可能忽略 `narrative_hint`。

**解决**：
- 在 system prompt 中强调"必须遵循叙述提示"
- 提供更具体的示例

## 🚀 未来优化

### **优化 1：时间线排序**

当前按 `published_at` 排序，可改进为：
- 识别事件的因果关系
- 按逻辑顺序（而非时间顺序）排列

### **优化 2：智能叙述模板**

根据新闻数量和类型，自动选择叙述模板：
- 2 条：对比式（"一边...另一边..."）
- 3 条：三段式（"首先...接着...最后..."）
- 4+ 条：总分式（"总体来看...具体到..."）

### **优化 3：跨主题关联**

当前只合并完全相同的主题，可扩展为：
- 识别上下游关系（如"芯片"和"手机"）
- 识别对比关系（如"苹果"和"华为"）
