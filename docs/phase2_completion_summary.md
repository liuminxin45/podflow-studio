### **2. 优化算法**

#### **阈值自适应**
```python
策略：
- 统计人工 reject 的主题平均分
- 平均分 + 0.5×标准差 作为新阈值
- 限制调整幅度，避免过度调整

置信度计算：
- 基于样本数：min(reject_count / 20, 1.0)
```

#### **权重动态优化**
```python
策略：
- 分析被拒绝主题在各维度的得分
- 如果某维度得分占最大值 > 50%，建议降低权重
- 优先调整影响最大的维度

置信度计算：
- 基于误报数：min(false_positive_count / 5, 1.0)
```

#### **Few-shot 生成**
```python
选择标准：
- 人工反馈有详细原因（+3分）
- 有标签（+1分/标签）
- 高度不一致（系统 must vs 人工 reject，+5分）
- 多样性：避免重复标签

排序：按质量评分倒序
```

### **优化建议**

#### **阈值调整**
```yaml
threshold_must_publish: 70.0 → 85.0 (+15.0)
threshold_maybe_publish: 55.0 → 75.0 (+20.0)

原因: 人工拒绝的主题平均分为 71.6±13.0
```

#### **权重调整（Top 5）**
```yaml

personal_impact_max: 10.0 → 8.5 (-1.5)    # 个人影响
continuity_max: 6.0 → 5.1 (-0.9)          # 连续性
data_enrichable_max: 6.0 → 5.1 (-0.9)     # 数据可补充
time_max: 5.0 → 4.2 (-0.8)                # 时效性
```

## 🚀 使用方式

### **基础使用**
```bash
# 分析反馈数据，生成优化建议
python tools/optimize_strategy.py

# 仅分析，不生成配置
python tools/optimize_strategy.py --dry-run

# 自动应用优化
python tools/optimize_strategy.py --auto-apply

# 指定输出路径
python tools/optimize_strategy.py \
  --report-file out/optimization_report.md \
  --feedback-dir feedback_history
```

### **应用优化**
```bash
# 方式 1: 手动应用
cp config/optimized_settings.yaml config/base/settings.yaml

# 方式 2: 自动应用
python tools/optimize_strategy.py --auto-apply
```

## ⚠️ 注意事项

### **1. 置信度问题**
- 当前只有 6 条反馈，置信度仅 30%
- **建议**: 至少收集 20+ 条反馈再应用优化

### **2. 过度拟合风险**
- 样本数过少可能导致过度拟合
- **建议**: 应用优化后持续监控效果

### **3. 阈值过高风险**
- 大幅提高阈值（+15~20分）可能导致漏报
- **建议**: 逐步调整，先提高 10 分观察效果
