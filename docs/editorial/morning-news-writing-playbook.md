# 早间资讯播客成稿方法

## 结论

两份范本最值得复用的不是固定口头禅，而是编辑顺序：先用具体变化抓住注意力，再替听众追问成本、门槛、风险、用途和限制，最后给一个有事实边界的判断。它们把“新闻发生了什么”推进到“这和我有什么关系”。

生产 Prompt 已落在 `nodes/script/prompts.py`：

- `build_episode_script_prompt`：生成完整节目。
- `build_quick_news_optimization_prompt`：在上下文中优化单条快讯。

## 范本文本量基线

两份范本的非空白字符分别为 5634 和 5693，结构分布也接近：开场约 400 字，快讯区约 2800 字，深度稿与收尾约 2450 字。这说明范本并不是 10 分钟、两三千字的短早报，而是约 22 分钟、五千多字的完整节目。

复算口径：样本为用户提供的 `优秀范本1.txt` 与 `优秀范本2.txt`；SHA-256 分别为
`8589252683a4450a97b1f3894e61d01b1653442f7251e41bc7c4024e00a64b53` 和
`85a3ca6a131e317f79e3823b4244a7412887ec6c842b605423e938697a324aba`。统计时按 Unicode
文本读取，并删除正则 `\s` 命中的全部空白字符后计数；原始字符数分别为 5884 和 5899。
结构区间由人工按开场、快讯、深度稿与收尾边界复核，因此属于编辑预算，不是逐段硬性校验值。

默认“范本节奏”据此设置为：

- 9 条快讯 + 1 条深度稿。
- 单条快讯 240–360 字。
- 深度稿 2000–2600 字。
- 开场 320–450 字，收尾 80–160 字。
- 单期 5200–6200 字，约 22 分钟。

短档采用 6 条快讯、2000–2800 字；加长档采用 10 条快讯 + 1 条深度稿、6500–7800 字。字数是素材充足时的编辑区间，素材不足时必须缩短，不能用重复内容或来源外信息补齐。

## 双表达体系

- **专业播报**：主持人退到信息之后，突出主体、出处、时间、数字和判断边界；删除口头禅、即时反应与购买倾向，不替听众做决定。
- **自然人味**：允许替听众追问、对具体数字做轻微反应，并使用少量自然口头衔接；每条最多两处，不随机添加“嗯、啊、其实”，不虚构亲历、采访或使用体验。

两套体系共享同一事实标准。预测、购买或投资看法必须有事实卡中的来源和条件，不能因为选择“自然人味”而放宽。文本中的人味还依赖 TTS 的停顿和韵律能力；语音模型表达有限时，应优先选择专业播报，避免把文字口癖机械读出。

## 从范本提取的节目结构

| 层级 | 范本中的有效做法 | 生产规则 |
| --- | --- | --- |
| 冷开场 | 连续抛出主体、动作和数字，快速建立本期信息密度 | 选 3 至 5 个最强信息点，不逐字念标题 |
| 深度预告 | 用连续问题制造“听完能得到答案”的承诺 | 问用途、成本、限制、真假辨别或普及距离，不用夸张悬念 |
| 节目仪式 | 主持人、日期、固定栏目帮助听众进入每日收听习惯 | 日期必须来自输入；“历史上的今天”没有事实卡就省略 |
| 快讯组 | 从灾情、安全等高时效信息，过渡到消费、科技和轻资讯 | 按紧急度和普遍相关性排序，素材不支持时不强行套顺序 |
| 中段重置 | 多条新闻后短暂报节目名，并再次提醒深度话题 | 一期最多一次，避免每条都用栏目口号打断 |
| 深度解读 | 从一个具象问题进入原理、玩家、价格、场景、限制和结论 | 每一层都必须有事实支撑；素材薄就缩短，不靠常识扩写 |
| 收尾 | 回收节目、提出听众问题、预告下一期 | 不重念摘要，不制造没有材料支撑的预告 |

## 单条快讯该讲什么

每条只围绕一个变化，按下面的优先级取材：

1. 谁在什么时间做了什么，最新变化是什么。
2. 哪一个至三个数字最能说明变化。
3. 谁会受到影响，影响发生在价格、资格、时间、安全还是选择上。
4. 听众现在能做什么，或者应该等哪一条后续信息。
5. 哪些内容仍是计划、预测、媒体说法或争议，不能写成定论。

不同题材对应不同的用户问题：

| 题材 | 听众真正关心的内容 |
| --- | --- |
| 灾害、安全 | 当前是否稳定、影响范围、什么时候必须绕行或停止操作、官方提醒 |
| 产品、服务 | 谁能用、价格、怎样开通、兼容哪些设备、什么时候上线 |
| 商业、投资 | 产品热度是否形成公司收入、谁真正受益、消费者会不会涨价；不要把票房或销量直接写成股价判断 |
| 科技 | 工作原理、已经能做到什么、价格或性能、续航和使用限制、离日常普及还有多远 |
| 招聘、旅行 | 资格、报名时间、真实成本、交通和履约条件、流量能否转化为长期供给 |
| 争议、维权 | 已知案例、品牌回应、检测或司法结论、举证难点；未建立因果时不能替任何一方定责 |

一条快讯不需要覆盖表中所有问题。编辑只选最能改变听众判断的一至两个问题。

## 深度解读为什么好听

范本会把大主题拆成一串递进问题。有效的问题链通常是：

1. 这个数字到底有多大，变化发生在哪里。
2. 它怎么做到，能否用一个普通场景解释。
3. 为什么行业现在需要它。
4. 已经有哪些产品、公司或路线，价格和性能差多少。
5. 普通人今天能用到什么，哪些仍在实验室或预测里。
6. 现阶段应当购买、尝试、谨慎还是观望。

每一问都要推动下一问。机制解释后接使用意义，行业规模后接真实出货或场景，未来判断前先讲限制。这样的衔接来自因果和用户决策，不依赖“首先、其次、最后”。

## 口播与衔接

- 一句话只承担一个主要动作。把主体和动词放在前面，把数字放在被解释的对象旁边。
- 长短句交替。多数句子应能一口气读完；公司名单、报告名和多层定语拆开写。
- 来源只在可信度、争议或时效需要时出现一次。连续“根据某某报道”会让节目像资料汇编。
- 新闻之间有共同人群、共同关键词或风险关系时顺势连接；没有关系就用一句短重置。
- 开场留下的深度问题必须在深度段逐一回答。不能只制造悬念。
- 删除空洞结尾，如“值得持续关注”“影响深远”。改成明确的后续变量、行动阈值或尚未确定的事项。

## 不应从范本继承的部分

两份文件像语音转写稿，存在重复词、错断句、节目名混写和英文名误识别。部分段落还把预测写得过满，或给出缺少材料闭环的购买与投资判断。生产稿必须做以下纠偏：

- 不复制转写口癖和语气词密度。
- 不把媒体预测、业内看法和公司计划写成已发生事实。
- 不用来源外的常识扩写技术原理、类比或市场规模。
- 不因“像范本”而加入固定的历史栏目、品牌口号或主持人姓名。
- 不把“用户关心”误解为必须给建议；事实不足时，最有用的内容是明确说出尚不能判断什么。

## 成稿质量门槛

以下四项是硬门槛，任一失败都应重写：

- 每个事实陈述能回到事实卡，确定性没有升级。
- 听众在前两句知道发生了什么。
- 每条快讯至少解决一个现实问题，而不是只改写标题。
- 文本可以直接朗读，没有网址、复杂括号、长串英文和资料清单腔。

建议按 100 分复核：事实与来源 30 分，听众价值 25 分，信息取舍 15 分，节目结构与衔接 15 分，口播节奏 10 分，语言自然度 5 分。低于 80 分不进入录制。

## 深度稿素材链路

整理页允许把一条新闻设为本期唯一深度稿。深度稿会提高到至少三个独立来源，并增加三类结构化整理：听众真正关心的问题、可展开的解释角度、现实价值与结论边界。自动研究使用 3–5 个可核验搜索问题，覆盖最新事实、形成机制、利益相关方、普通人门槛和反方证据。

普通快讯仍使用精简事实卡；被指定的深度稿则把整理后的证据包保留在现有 summary 字符串字段中，最多 6000 字符，不改变共享协议 schema。即便如此，2000–2600 字仍只是素材充足时的建议区间：来源不足、角度重复或关键数字无法核验时必须缩短，不能让成稿模型用常识或想象补齐。
