# 测试清理功能使用指南

## 📋 概述

Auto-Podcast项目现在支持智能的测试产物管理功能，可以根据测试结果自动清理或保留测试产物文件。

## 🎯 功能特性

### ✅ 智能清理规则
- **测试全部成功**: 自动清理所有生成的测试产物文件
- **测试部分失败**: 保留所有产物文件用于调试
- **强制保留模式**: 始终保留产物文件，忽略测试结果

### 📁 管理的测试产物
- TTS测试文件：`test_text.txt`, `test_ssml.ssml`, `test_output.mp3` 等
- LLM测试文件：`test_prompts.txt`, `test_config.json` 等
- Research测试文件：配置文件和临时数据
- Fetch测试文件：输出数据和配置文件

## 🚀 使用方式

### 1. 默认模式（自动清理）
```bash
# 运行所有测试，成功后自动清理
python run_tests_enhanced.py

# 运行单个模块测试
python src/tts/tests/test_tts_client.py
```

**行为**：
- ✅ 所有测试通过 → 自动清理产物
- ❌ 部分测试失败 → 保留产物用于调试

### 2. 保留模式（不自动清理）
```bash
# 运行测试但保留所有产物
python run_tests_enhanced.py --no-cleanup

# 运行单个测试但保留产物
python src/tts/tests/test_tts_client.py --no-cleanup
```

**行为**：
- ✅ 始终保留所有测试产物文件
- 📁 产物文件列表会在测试结束后显示

### 3. 交互式模式
```bash
# 交互式选择测试模块
python run_tests_enhanced.py --interactive
```

## 🛠️ 技术实现

### 核心组件
- **TestResultManager**: 测试结果和产物管理器
- **test_utils.py**: 测试工具模块
- **run_tests_enhanced.py**: 增强版测试运行器

### 清理逻辑
```python
# 伪代码示例
if auto_cleanup and all_tests_passed:
    cleanup_artifacts()
elif auto_cleanup and some_tests_failed:
    keep_artifacts_for_debugging()
else:  # no_cleanup mode
    keep_all_artifacts()
```

## 🔧 自定义配置

### 修改清理规则
可以在 `src/utils/test_utils.py` 中自定义：

```python
class TestResultManager:
    def __init__(self, auto_cleanup=True, ...):
        self.auto_cleanup = auto_cleanup
    
    def get_test_artifacts(self):
        # 自定义要管理的文件类型
        artifacts = [...]
        return artifacts
```

### 添加新的测试产物类型
```python
# 在 get_test_artifacts() 中添加
new_test_data = self.project_root / "src" / "new_module" / "tests" / "test_data"
if new_test_data.exists():
    artifacts.extend([
        new_test_data / "custom_output.txt",
        new_test_data / "custom_config.json"
    ])
```

## 🎯 优势

1. **🧹 环境整洁**: 成功测试后自动清理，避免文件堆积
2. **🔍 调试友好**: 失败时保留产物，便于问题分析
3. **⚙️ 灵活配置**: 支持多种运行模式
4. **📊 清晰反馈**: 明确显示清理状态和文件列表
5. **🚀 自动化**: 减少手动清理工作

## 📞 故障排除

### 清理失败
如果出现清理失败，检查：
- 文件权限是否正确
- 文件是否被其他程序占用
- 磁盘空间是否充足

### 产物文件过多
如果产物文件积累过多：
```bash
# 手动清理所有测试产物
python -c "
from src.utils.test_utils import cleanup_test_artifacts
print(f'清理了 {cleanup_test_artifacts()} 个文件')
"
```
