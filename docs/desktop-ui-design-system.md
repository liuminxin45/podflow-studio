# PodFlow Studio 桌面端 UI 设计系统

状态：**生效中**

适用范围：Electron / React 工作台
最后更新：2026-07-15

## 设计基线

PodFlow Studio 是高密度的创作工作台。界面层级应由布局、留白、
色彩和位置建立，而不是反复使用粗体、重阴影或高对比色块。

- 默认采用 **400** 字重；普通标签、导航与次级操作采用 **500**。
- 区块标题、列表标题和强语义文本采用 **550**。
- 页面标题和唯一的主操作采用 **600**；不使用 650、700、750、800
  作为常规界面强调。
- 选中态使用浅灰底色或细描边；不要同时叠加重描边、阴影和位移动画。
- 容器优先使用白底、`1px` 细描边和 `5–10px` 圆角；常规容器不使用阴影。
- 颜色表达语义，而非装饰：仅成功、警告、错误和主操作使用语义色。

## 令牌

`src/index.css` 的 **Quiet workbench pass** 和 `src/App.tsx` 的
Ant Design `ConfigProvider` 是运行时令牌的唯一来源。新增界面必须使用
这些变量或对应的 Ant Design token，不能另建一套颜色、圆角或阴影体系。

| 类别 | 令牌 | 值 / 用途 |
| --- | --- | --- |
| 画布 | `--bg-primary` | `#ffffff`，默认工作区底色 |
| 容器 | `--bg-secondary` | `#ffffff`，侧栏与内容面 |
| 悬停 | `--bg-tertiary` | `#f6f6f5`，弱反馈 |
| 选中 | `--accent-light` | `#f0f1f1`，非破坏性选中态 |
| 文本 | `--text-primary` | `#242629`，主内容 |
| 次文本 | `--text-secondary` | `#686a6d`，说明与次要信息 |
| 边框 | `--border-color` | `#e6e7e7`，常规分隔 |
| 活动边框 | `--border-active` | `#c8cbcd`，焦点/选中辅助反馈 |
| 主操作 | `--accent-primary` | `#303438`，唯一主按钮与关键状态 |
| 圆角 | `--radius-sm/md/lg` | `5px / 7px / 10px` |
| 阴影 | `--shadow-soft` | 仅极弱层次；普通容器为 `none` |

## 组件规则

### 导航与列表

- 每行只保留一个主要状态：浅色背景、细边框或左侧指示条三选一。
- 图标按钮必须有 `title` 和 `aria-label`；有明确目的的入口优先采用
  “左图标 + 右文字”的导航项。
- 完成、警告等状态用小尺寸标签表达，不能把整行染成高饱和色。

### 按钮、表单与卡片

- 普通按钮字重 500，主按钮 600；同一视觉区域最多一个主按钮。
- 输入控件高度为 `32px`，紧凑尺寸为 `28px`。
- Card、Modal、Drawer、Popover 使用细边框，无投影；不要把每一块信息
  都包装成卡片。

### 动效与可访问性

- 悬停反馈只改变背景或边框；不使用上浮、缩放或脉冲作为常规反馈。
- 遵守 `prefers-reduced-motion`，动效必须可被降级。
- 不能只靠颜色表达状态；保留文字、图标或形状提示。

## 防漂移流程

1. 新增/修改 UI 前，先阅读本文件和 `src/index.css` 的 Quiet workbench
   pass。
2. 使用现有令牌；若确需新增令牌，同时更新本文件、`src/index.css`、
   `src/App.tsx` 的 Ant Design token，以及守卫脚本。
3. 不要在 Quiet workbench pass 之后追加样式规则；需扩展时直接修改该
   区块，使其始终保有最终覆盖权。
4. 提交前运行 `npm run verify:ui-system`，再运行 `npm run build`。
5. 涉及交互流程的页面，再运行 `npm run acceptance:cdp`。

`npm run verify:offline` 会自动包含 UI 设计系统守卫。守卫验证令牌、
层级规则、无障碍降级规则和这份文档的存在，避免样式基线被无意移除。
